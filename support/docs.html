Concepts

  • Tenant — isolated environment for a district/institution.
  • Governed dataset — subject to masking & row policies, row-level ABAC.
  • Promotion pipeline — raw → canonical → publish, gated by DQ approvals.

Onboarding

  1. Admin configures SSO/SCIM and user/role assignments.
  2. Engineer sets up connectors & mappings (Ed-Fi / OneRoster / CEDS).
  3. Steward defines DQ rules, validates, and approves promotions.

Governance

Classification tags (e.g., PII_STRICT), masking policies, and ABAC row filters apply across SQL, API, exports, and shares. All actions are immutably audited.

Connectors

Use the New Connector Wizard with credential vault integration. Retries, throttling, and quarantine dead-letter queues are built in.

APIs

EDX exposes read APIs for governed publish views. See the API Reference for endpoints and schemas.

Exports & Shares

Consumers request exports and can access secure, revocable shares. All masking & row filters apply.
